<title>Introduction</title>
<p id="_f46875e0-7766-7d01-a44d-4c3d9f1155fa">The W3C Semantic Web Activity is defining a collection of technologies that enables a “web of data” where information is easily shared and reused across applications. 000608_core
Hanging paragraph in clause
<clause id="_core" obligation="normative">
<title>Core</title>
<p id="core">This clause establishes the <strong>Core</strong> requirements class, with IRI <tt>/req/core</tt>, which has a single corresponding conformance class, <strong>Core</strong>, with IRI <tt>/conf/core</tt>. This requirements class defines a set of classes and properties for representing geospatial data. The resulting vocabulary — an ontology — can be used to construct SPARQL graph patterns for querying appropriately modeled geospatial data. The RDFS and OWL vocabularies have both been used so that the vocabulary can be understood by systems that support only RDFS entailment and by systems that support OWL-based reasoning.</p>

<p id="_8d6b1344-c266-f4a8-fcce-d56d863fbbb1">The figure below gives an overview of the classes and properties defined by GeoSPARQL in the <strong>Core</strong>, <strong>Topology Vocabulary Extension</strong> and <strong>Geometry Extension</strong>, <strong>Geometry Topology Extension</strong> and <strong>RDFS Entailment Extension</strong> Conformance Classes.</p>
